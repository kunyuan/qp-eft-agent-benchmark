# Task 04: Generic Quasiparticle Runner

Goal: implement the runner used by the hidden evaluator.

The required interface is:

```bash
python run_qp.py --element-config element_config.json --grid grid.csv --out qp_bands.csv
```

The runner must write:

```text
element,point_id,t,E_pred_eV
```

Rules:

- It must accept any element config with the documented schema.
- It must not hardcode Na or Al output values.
- It must not inspect ARPES reference files.
- It must output all occupied bands at each requested point.
- It must use the same code path for public and hidden elements.

Recommended architecture:

```text
run_qp.py
  parse CLI
  load element_config.json
  load grid.csv
  compute/load Kohn-Sham occupied bands
  compute frozen-core correction
  write qp_bands.csv
```

If your DFTK workflow is expensive, cache Kohn-Sham outputs by element and
document the cache in `method.md`. The hidden evaluator may run in a clean
directory, so any cache required for hidden elements must be generated by the
runner itself or derived from runtime inputs.
